Connect4 Reinforcement Learning

Training Pipeline

Self-play positions are searched with Monte Carlo Tree Search, labeled with visit-count policies and terminal outcomes, stored in the replay buffer, and used to optimize a dual-head convolutional network.
1. State Encoding

Board to Tensor

Each board is converted to a 3 × 6 × 7 tensor: empty cells, player 1 discs, and player 2 discs.

Input to network: current position only, no handcrafted features.
2. Search

MCTS Rollouts

For each move, the agent runs repeated selection, expansion, evaluation, and backpropagation from the root.

Child choice uses prior-guided UCB with visit counts and node value estimates.
3. Targets

Policy + Value Labels

The move target is the normalized visit distribution; the value target is the final game outcome from the acting player’s perspective.

Value targets are discrete: +1 win, 0 draw, -1 loss.
4. Optimization

Supervised Update

The network minimizes policy KL divergence plus value MSE over batches sampled from the replay data.

Optimizer: Adam, gradient clipping: 1.0, scheduler: StepLR.
5. Evaluation

Progress Score

Periodic evaluation measures self-play balance and win rate versus a random player, then aggregates them into a custom score.

This score is a project-specific metric, not Elo.

Search Rule

UCB(s, a) = Q(s, a) + cpuct P(s, a) √N(s) / (1 + N(s, a))
Implemented in mcts_components/calculate_ucb.py with c_puct = 1.4.

Training Objective

L = KL(πMCTS || πθ) + MSE(v, z)
Implemented in neural_network_components/calculate_loss.py.
