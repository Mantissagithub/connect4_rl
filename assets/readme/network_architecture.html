Connect4 Neural Network

Dual-Head Convolutional Architecture

The model maps a 3-channel board tensor to both a move distribution over the 7 columns and a scalar value in [-1, 1]. The implementation uses a compact shared trunk followed by lightweight policy and value heads.

Input

Board Tensor
3 × 6 × 7
Channel 0: empty cells
Channel 1: player 1 discs
Channel 2: player 2 discs

Shared Trunk

Conv Block 1
3 → 32
3 × 3 conv, padding 1, ReLU
Conv Block 2
32 → 64
3 × 3 conv, padding 1, ReLU
Conv Block 3
64 → 128
3 × 3 conv, padding 1, ReLU

Policy Head

1 × 1 Reduction
128 → 2
Preserves 6 × 7 spatial map
Flatten
2 × 6 × 7 = 84
Concatenate reduced features
Linear + Softmax
84 → 7
One probability per legal column

Outputs π(a|s), the move distribution used by MCTS as a prior and by training as the policy prediction.

Value Head

1 × 1 Reduction
128 → 1
Preserves 6 × 7 spatial map
Flatten + Hidden Layer
42 → 64
Linear, ReLU
Output
64 → 1
Linear, tanh

Outputs v(s), the estimated game outcome from the current state, bounded to [-1, 1].

Parameter Count

97,047 trainable parameters across the three shared convolutional layers and the two output heads.

Policy Target

The policy head is trained against the normalized MCTS visit distribution using KL divergence.

Value Target

The value head is trained against the final game result using mean squared error.
